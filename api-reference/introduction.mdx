Добро пожаловать в документацию по API Квант! Здесь вы найдете информацию обо всех конечных точках API Квант, сможете ознакомиться с инструкциями по их использованию и узнать, какие данные они возвращают.

API Квант разработан с учетом архитектурного стиля REST, предлагает логично структурированные URL, предназначенные для работы с ресурсами, принимает запросы с данными в формате `application/x-www-form-urlencoded`, отвечает данными в формате JSON, а также следует стандартам использования HTTP статус-кодов, механизмов аутентификации и HTTP методов.

## Использование API

REST API Квант разработан так, чтобы его было просто использовать. Вы можете выбрать любой язык и платформу для отправки запросов. Чтобы лучше ознакомиться с API, вы также можете использовать инструменты вроде [Postman](https://www.postman.com/) или [Insomnia](https://mintlify.com/docs/api-playground/mdx/configuration).

## Аутентификация

API Квант использует API-ключи для проверки и авторизации запросов. Вы можете управлять и просматривать свои API-ключи через панель управления Квант, или запросить ключ через поддержку. Вам нужно передать свой API-ключ в заголовке авторизации с префиксом `Bearer` при отправке запроса.

```json
Bearer ВАШ_API_КЛЮЧ
```
API-ключи - это ключи к вашему аккаунту, поэтому их нужно хранить в тайне и безопасности. Не делитесь ими на GitHub или где-либо в интернете, где кто-то мог бы их увидеть.

Также помните, что эти ключи предназначены для использования на вашем сервере, а не в клиентском коде, который виден всем пользователям вашего сайта. Это мера предосторожности, чтобы никто случайно не увидел ваш ключ и не получил доступ к вашему аккаунту.
