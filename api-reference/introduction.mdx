Добро пожаловать в документацию по API Квант. Здесь вы найдете информацию обо всех методах, сможете ознакомиться с инструкциями по их использованию и узнать, какие данные они возвращают.

API Квант разработан с учетом архитектурного стиля REST, предлагает логично структурированные URL, предназначенные для работы с ресурсами, принимает запросы с данными в формате `application/json`, отвечает данными в формате JSON, а также следует стандартам использования HTTP статус-кодов, механизмов аутентификации и HTTP методов.

## Использование API

REST API Квант разработан так, чтобы его было просто использовать. Чтобы лучше ознакомиться с API, вы также можете использовать инструменты вроде [Postman](https://www.postman.com/) или [Insomnia](https://insomnia.rest/).

## Аутентификация

API Квант использует API-ключи для проверки и авторизации запросов. Вы можете управлять и просматривать свои API-ключи через панель управления Квант или запросить ключ через поддержку. 

Для отправки запросов к API Квант, используйте базовый URL: `https://api.kvant.app/`. Все запросы должны быть отправлены на этот адрес с соответствующими путями и параметрами.

При отправке запроса, необходимо передать ваш API-ключ в заголовке `Authorization` с префиксом `Bearer`:

```json
Authorization: Bearer ВАШ_API_КЛЮЧ
```
API-ключи - это ключи к вашему аккаунту, поэтому их нужно хранить в тайне и безопасности. Не делитесь ими на GitHub или где-либо в интернете, где кто-то мог бы их увидеть.

Также помните, что эти ключи предназначены для использования на вашем сервере, а не в клиентском коде, который виден всем пользователям вашего сайта. Это мера предосторожности, чтобы никто случайно не увидел ваш ключ и не получил доступ к вашему аккаунту.

Детальные примеры использования каждого метода доступны на соответствующих страницах документации к каждому методу API.
