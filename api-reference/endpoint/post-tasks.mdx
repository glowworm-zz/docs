---
title: 'Создание коммуникации'
openapi: 'POST /api/tasks/'
---

Метод `POST /api/tasks` позволяет создать новую задачу в системе Квант. Необходимо передать в теле запроса данные задачи.
Вы должны указать название задачи и исполнителя, но вы также можете определить тип задачи и описание.

Если вы не зададите тип коммуникации при создании, будет использован тип по умолчанию. Эти параметры включают "задачу", "обращение", "принятое решение".

Политика доступа к задаче устанавливается как открытая по умолчанию для новых задач. Вы также можете добавить политику доступа при создании новой задачи, добавив `"accessPolicy": { "type": "public" }`

### Авторизация

Для всех запросов к API требуется API-ключ. Вы должны передать его в заголовке Authorization с префиксом Bearer.

```json
Bearer ваш_API-ключ
```

### Тело запроса

Тело запроса должно быть в формате `application/json` и содержать следующие поля:

```json
{
  "name": "Название задачи",
  "type": "тип задачи",
  "description": "Описание задачи",
  "to_user_id": "идентификатор пользователя, назначенного исполнителем"
}
```

- `name` (string, required) - Название задачи.
- `type` (string, required) - Тип задачи, допустимые значения: "задача", "обращение", "принятое решение".
- `description` (string, optional) - Подробное описание задачи.
- `to_user_id` (string, required) - Идентификатор пользователя, назначенного исполнителем.

### Успешный ответ
При успешном создании задачи возвращается объект с подробной информацией о задаче.

```json
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "идентификатор задачи",
  "name": "Название задачи",
  "type": "тип задачи",
  "status": "входящая",
  "description": "Описание задачи",
  "to_user_id": "идентификатор пользователя, назначенного исполнителем",
  "created_at": "дата и время создания задачи в формате ISO 8601"
}
```

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

```json
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Ошибка валидации",
  "message": "Неверно указаны данные задачи"
}
```
### Пример использования
Для создания задачи отправьте POST запрос на `/api/tasks` с необходимыми данными в JSON формате. В ответе вы получите информацию о созданной задаче или описание ошибки, если запрос выполнить не удалось.
