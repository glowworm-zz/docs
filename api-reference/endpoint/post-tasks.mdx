---
title: 'Создание коммуникации'
openapi: 'POST /tasks/'
---

Метод `POST /tasks` позволяет создать новую задачу в системе Квант. Необходимо передать в теле запроса данные задачи.
Вы должны указать название задачи и исполнителя, но вы также можете определить тип задачи и описание.

Если вы не зададите тип коммуникации при создании, будет использован тип по умолчанию. Эти параметры включают "задачу", "обращение", "принятое решение".

Политика доступа к задаче устанавливается как открытая по умолчанию для новых задач. Вы также можете добавить политику доступа при создании новой задачи, добавив `"accessPolicy": { "type": "public" }`

### Параметры запроса

- `name` (string, required) - Название задачи.
- `type` (string, optional) - Фильтр по типу задачи.  
  Допустимые значения: `task` (задача), `request` (обращение), `decision` (решение), `meeting` (встреча).
- `description` (string, optional) - Описание задачи.
- `performer` (number, required) - Идентификатор пользователя, назначенного исполнителем.

```typescript
// POST /tasks
interface Request { 
  name: "string",
  type?: "string",
  description?: "string",
  performer: number
}
```

### Пример запроса
<CodeGroup>

```md Request
{
  "name": "Разработка API",
  "type": "task",
  "description": "Разработать новый API для системы Квант",
  "performer": 12345
}
```

```bash cURL
curl -X POST "https://api.kvant.app/tasks" \
     -H "Content-Type: application/json" \
     -d '{
           "name": "Разработка API",
           "type": "task",
           "description": "Разработать новый API для системы Квант",
           "performer": 12345
         }'
```

</CodeGroup>

### Успешный ответ
При успешном создании задачи возвращается объект с информацией о задаче.

```json
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 67890,
  "name": "Разработка API",
  "type": "task",
  "status": "incoming",
  "description": "Разработать новый API для системы Квант",
  "performer": 12345,
  "created_at": "2024-12-05T12:00:00Z"
}
```

### Описание элементов ответа

- `id` (number) - Уникальный идентификатор задачи.
- `status` (string, optional) - Фильтр по статусу задачи.  
  Допустимые значения: `incoming` (входящие), `accepted` (принятые), `in_progress` (в работе), `completed` (выполненные), `planned` (запланированные), `on_approval` (на утверждении).
- `created_at` (string) - Дата и время создания задачи в формате ISO 8601.

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

- 400 (Bad Request):
```json
{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса. Пожалуйста, проверьте, что все обязательные поля заполнены корректно и повторите попытку."
}
```

- 401 (Unauthorized):
```json
{
  "error": "Неавторизован",
  "message": "Необходима аутентификация. Убедитесь, что вы предоставили правильный API-ключ в заголовке запроса."
}
```
- 403 (Forbidden):
```json
{
  "error": "Доступ запрещен",
  "message": "У вас нет прав для создания новой задачи. Пожалуйста, обратитесь к администратору системы для получения соответствующих разрешений."
}
```
- 404 (Not Found):
```json
{
  "error": "Задача не найдена",
  "message": "Задача с указанным ID не существует. Убедитесь, что вы используете правильный идентификатор задачи."
}
```
- 429 (Too Many Requests):
```json
{
  "error": "Слишком много запросов",
  "message": "Превышен лимит запросов. Подождите некоторое время, прежде чем отправлять новый запрос."
}
```
- 500 (Internal Server Error):
```json
{
  "error": "Внутренняя ошибка сервера",
  "message": "Произошла ошибка на сервере при обработке вашего запроса. Пожалуйста, попробуйте еще раз позже или обратитесь в службу поддержки."
}
```
