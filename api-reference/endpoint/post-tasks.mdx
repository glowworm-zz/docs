---
title: 'Создание коммуникации'
openapi: 'POST /api/tasks/'
---

Метод `POST /api/tasks` позволяет создать новую задачу в системе Квант. Необходимо передать в теле запроса данные задачи.
Вы должны указать название задачи и исполнителя, но вы также можете определить тип задачи и описание.

Если вы не зададите тип коммуникации при создании, будет использован тип по умолчанию. Эти параметры включают "задачу", "обращение", "принятое решение".

Политика доступа к задаче устанавливается как открытая по умолчанию для новых задач. Вы также можете добавить политику доступа при создании новой задачи, добавив `"accessPolicy": { "type": "public" }`

### Параметры запроса

- `name` (string, required) - Название задачи.
- `type` (string, optional) - Фильтр по типу задачи.  
  Допустимые значения: `task` (задача), `request` (обращение), `decision` (решение), `meeting` (встреча).
- `description` (string, optional) - Описание задачи. Это поле не обязательно для заполнения.
- `to_user_id` (number, required) - Идентификатор пользователя, назначенного исполнителем.

### Тело запроса

Тело запроса должно быть в формате `application/json` и содержать следующие поля:

```json
{
  "name": "string",
  "type": "string",
  "description"?: "string",
  "to_user_id": number
}
```

### Пример запроса
```json
{
  "name": "Разработка API",
  "type": "task",
  "description": "Разработать новый API для системы Квант",
  "to_user_id": 12345
}
```

### Успешный ответ
При успешном создании задачи возвращается объект с информацией о задаче.

```json
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 67890,
  "name": "Разработка API",
  "type": "task",
  "status": "incoming",
  "description": "Разработать новый API для системы Квант",
  "to_user_id": 12345,
  "created_at": "2023-12-05T12:00:00Z"
}
```

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

- 400 (Bad Request):
```json
{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```

- 401 (Unauthorized):
```json
{
  "error": "Неавторизован",
  "message": "Необходима аутентификация"
}
```
- 403 (Forbidden):
```json
{
  "error": "Доступ запрещен",
  "message": "У вас нет доступа к этой задаче"
}
```

- Ошибка 429 (Too Many Requests):
```json
{
  "error": "Слишком много запросов",
  "message": "Превышен лимит запросов"
}
```
