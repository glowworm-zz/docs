---
title: 'Информация о коммуникации'
openapi: 'GET /api/tasks/{id}'
---
Метод `GET /api/tasks/{id}` позволяет получить детальную информацию о конкретной коммуникации по её идентификатору. Этот метод полезен, когда вам нужны подробные данные об одной задаче, включая ее статус, тип, описание и другую информацию.

### Параметры запроса

Для получения информации о задаче, необходимо указать идентификатор задачи в пути запроса:

- `id` (number, required) - уникальный идентификатор задачи.

### Пример запроса
```json
GET /api/tasks/12345
Authorization: Bearer ваш_API_ключ
```

### Успешный ответ
При успешном запросе сервер вернет объект, содержащий информацию о задаче.

```json
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 12345,
  "name": "Обновление системы",
  "type": "task",
  "status": "in_work",
  "description": "Обновление программного обеспечения до последней версии",
  "to_user_id": 67890,
  "created_at": "2023-12-05T10:00:00Z"
}
```

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

- 400 (Bad Request):
```json
{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```

- 401 (Unauthorized):
```json
{
  "error": "Неавторизован",
  "message": "Необходима аутентификация"
}
```
- 403 (Forbidden):
```json
{
  "error": "Доступ запрещен",
  "message": "У вас нет доступа к этой задаче"
}
```
- 404 (Not Found):
```json
{
  "error": "Задача не найдена",
  "message": "Задача с указанным ID не существует"
}
```
- Ошибка 429 (Too Many Requests):
```json
{
  "error": "Слишком много запросов",
  "message": "Превышен лимит запросов"
}
```
