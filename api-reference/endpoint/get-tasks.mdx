---
title: 'Список коммуникаций'
openapi: 'GET /api/tasks/'
---
Метод `GET /api/tasks` предоставляет доступ к списку коммуникаций в системе Квант. Этот метод позволяет извлекать данные о всех созданных задачах, с возможностью фильтрации.

Вы можете фильтровать коммуникации по различным параметрам, включая тип задачи, статус (например, "входящие", "принятые", "в работе", "выполненные", "запланированные") и назначенному исполнителю.

### Парметры запроса

Этот метод поддерживает следующие параметры запроса для фильтрации списка коммуникаций:

- `type` (string, optional) - Фильтр по типу задачи.  
  Допустимые значения: `task` (задача), `request` (обращение), `decision` (решение), `meeting` (встреча).
- `status` (string, optional) - Фильтр по статусу задачи.  
  Допустимые значения: `incoming` (входящие), `accepted` (принятые), `in_progress` (в работе), `completed` (выполненные), `planned` (запланированные).
- `to_user_id` (string, optional) - Фильтр по идентификатору исполнителя.
- `total_count` (number) - Общее количество задач в списке, возвращаемое запросом.

```json
GET /api/tasks
{
  "type"?: "task" | "request" | "decision" | "meeting",
  "status"?: "incoming" | "accepted" | "in_progress" | "completed" | "planned",
  "to_user_id"?: string
}
```
### Пример запроса
```json
GET /api/tasks?type=task&status=incoming&to_user_id=123
```

### Успешный ответ
Сервер вернет массив объектов, каждый из которых представляет задачу.

```json
{
  "total_count": 2,
  "tasks": [
    {
      "id": 67890,
      "name": "Изучить API Кванта",
      "type": "task",
      "status": "in_progress",
      "description": "Подробно изучить документацию API Кванта и применить знания на практике",
      "to_user_id": 123,
      "created_at": "2023-12-05T12:00:00Z"
    },
    {
      "id": 67891,
      "name": "Реализовать интеграцию с Квант",
      "type": "task",
      "status": "planned",
      "description": "Спроектировать и реализовать интеграцию с Квант для улучшения рабочих процессов",
      "to_user_id": 123,
      "created_at": "2023-12-10T10:00:00Z"
    }
  ]
}
```

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

- 400 (Bad Request):
```json
{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```

- 401 (Unauthorized):
```json
{
  "error": "Неавторизован",
  "message": "Необходима аутентификация"
}
```
- О400 (Bad Request):
```json
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```
- 400 (Bad Request):
```json
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```
- Ошибка 400 (Bad Request):
```json
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
```
