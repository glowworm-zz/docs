---
title: 'Список коммуникаций'
openapi: 'GET /api/tasks/'
---
Метод `GET /api/tasks` предоставляет доступ к списку коммуникаций в системе Квант. Этот метод позволяет извлекать данные о всех созданных задачах, с возможностью фильтрации.

Вы можете фильтровать коммуникации по различным параметрам, включая тип задачи, статус (например, "входящие", "принятые", "в работе", "выполненные", "запланированные") и назначенному исполнителю.

### Парметры запроса

Этот метод поддерживает следующие параметры запроса для фильтрации списка коммуникаций:

- `type` (string, optional) - Фильтр по типу задачи.
- `status` (string, optional) - Фильтр по статусу задачи.
- `to_user_id` (string, optional) - Фильтр по идентификатору исполнителя.


### Успешный ответ
Сервер вернет массив объектов, каждый из которых представляет задачу.

```json
HTTP/1.1 200 OK
Content-Type: application/json
{
[
  {
    "id": number, // ID задачи
    "name": string, // Название задачи
    "type": "task" | "request"| "decition"| "meeting", // Тип задачи
    "status": "task" | "request"| "decition"| "meeting", // Статус задачи
    "description": string, // Описание задачи
    "to_user_id": "идентификатор исполнителя",
    "created_at": "дата и время создания задачи в формате ISO 8601"
  }
  ]
  }
```

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

```json
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса"
}
```
### Пример использования
Для получения списка коммуникаций отправьте GET запрос на `/api/tasks` с необходимыми параметрами фильтрации. Например, чтобы получить список задач, назначенных определенному пользователю, ваш запрос может выглядеть так:

```json
GET /api/tasks?to_user_id=идентификатор_пользователя
```

В ответе вы получите список задач, соответствующих заданным критериям фильтрации.
