---
title: 'Список коммуникаций'
openapi: 'GET /api.kvant.app/tasks/'
---
Метод `GET /api.kvant.app/tasks` предоставляет доступ к списку коммуникаций в системе Квант. Этот метод позволяет извлекать данные о всех созданных задачах, с возможностью фильтрации.

Вы можете фильтровать коммуникации по различным параметрам, включая тип задачи, статус (например, "входящие", "принятые", "в работе", "выполненные", "запланированные", "на утверждении") и назначенному исполнителю.

### Параметры запроса

Этот метод поддерживает следующие параметры запроса для фильтрации списка коммуникаций:

- `type` (string, optional) - Фильтр по типу задачи.  
  Допустимые значения: `task` (задача), `request` (обращение), `decision` (решение), `meeting` (встреча).
- `status` (string, optional) - Фильтр по статусу задачи.  
  Допустимые значения: `incoming` (входящие), `accepted` (принятые), `in_progress` (в работе), `completed` (выполненные), `planned` (запланированные), `on_approval` (на утверждении).
- `performer` (number, optional) - Фильтр по исполнителю.
- `total_count` (number) - Общее количество задач в списке, возвращаемое запросом.

```typescript
// GET /api.kvant.app/tasks
interface Request { 
  type?: "task" | "request" | "decision" | "meeting"
  status?: "incoming" | "accepted" | "in_progress" | "completed" | "planned" | "on_approval"
  performer?: number
}
```

### Пример запроса

<CodeGroup>

```md Request
GET /api/tasks?type=task&status=incoming&performer=123
```

```bash cURL
curl -X GET "https://api.kvant.app/tasks?status=in_progress&type=task&performer=123"
```

</CodeGroup>

### Успешный ответ
Сервер вернет массив объектов, каждый из которых представляет задачу.

```json
{
  "total_count": 2,
  "tasks": [
    {
      "id": 67890,
      "name": "Изучить API Кванта",
      "type": "task",
      "status": "in_progress",
      "description": "Подробно изучить документацию API Кванта и применить знания на практике",
      "performer": 123,
      "created_at": "2024-12-05T12:00:00Z"
    },
    {
      "id": 67891,
      "name": "Реализовать интеграцию с Квант",
      "type": "task",
      "status": "planned",
      "description": "Спроектировать и реализовать интеграцию с Квант для улучшения рабочих процессов",
      "performer": 123,
      "created_at": "2024-12-10T10:00:00Z"
    }
  ]
}
```

### Описание элементов ответа

- `id` (number) - Уникальный идентификатор задачи.
- `name` (string) - Название задачи.
- `description` (string) - Описание задачи.
- `created_at` (string) - Дата и время создания задачи в формате ISO 8601.

### Ошибка
Если запрос не может быть обработан из-за неверных данных, возвращается ошибка с соответствующим описанием.

- 400 (Bad Request):
```json
{
  "error": "Ошибка в запросе",
  "message": "Неверные параметры запроса. Проверьте правильность всех введенных данных, особенно фильтров."
}
```

- 401 (Unauthorized):
```json
{
  "error": "Неавторизован",
  "message": "Необходима аутентификация. Убедитесь, что вы предоставили правильный API-ключ в заголовке запроса."
}
```
- 403 (Forbidden):
```json
{
  "error": "Доступ запрещен",
  "message": "У вас нет доступа к этой задаче. Проверьте, что у вашего аккаунта есть соответствующие права доступа."
}
```
- 404 (Not Found):
```json
{
  "error": "Задача не найдена",
  "message": "Задача с указанным ID не существует. Убедитесь, что вы используете правильный идентификатор задачи."
}
```
- Ошибка 429 (Too Many Requests):
```json
{
  "error": "Слишком много запросов",
  "message": "Превышен лимит запросов. Подождите некоторое время, прежде чем отправлять новый запрос."
}
```
- 500 (Internal Server Error):
```json
{
  "error": "Внутренняя ошибка сервера",
  "message": "Произошла ошибка на сервере при обработке вашего запроса. Пожалуйста, попробуйте еще раз позже или обратитесь в службу поддержки."
}
```
